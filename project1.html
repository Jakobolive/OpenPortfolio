Project 1: Slick Weather App

The Slick Weather App, developed using Flutter and Dart, aimed to provide a seamless and user-friendly experience for displaying current weather conditions and a 5-day forecast. The project involved integrating live weather APIs, utilizing device location data, and implementing animations to create a visually engaging application. The app's goals included accurate location-based weather retrieval, efficient API handling, and a clean, intuitive interface that highlights essential information without overwhelming the user.

The development process was structured into three key components: service and API calls, the current weather display, and the 5-day forecast display. This modular design approach facilitated a deeper understanding of each component's requirements, ensuring clarity and reducing potential confusion during implementation.

Challenges included managing user permissions for accessing location data, organizing API responses for optimal display, and incorporating animations to enhance the app's visual appeal. These obstacles provided valuable lessons, such as the importance of concise data presentation for mobile devices and the effective use of design elements to improve user experience.

Overall, the project successfully met its objectives and serves as a strong foundation for further enhancements, demonstrating the versatility of Flutter and Dart in mobile application development.

Key Features

  • Feature 1: Fetch Users Current Location, Latitude And Longitude
  • Feature 2: Call A Weather API Using Current Location
  • Feature 3: Call A City API Using Latitude And Longitude
  • Feature 4: Determine Date & Weather Animation To Display
  • Feature 5: Display Current & 5 Day Forecast On The Screen Including Animation

Technologies Used

Dart, Flutter.

Honest Reflection

This project served as a great conclusion to the mobile development course, presenting both design challenges and practical experience with real API integrations. Unlike the mock API files used earlier in the course, working with live APIs for weather and city data highlighted the challenges of real-world API calls apart from JSON file reads. The development process was smooth and structured, with the application design segmented into three main components: service and API calls, the current weather display, and the 5-day forecast display. This division within the app design produced a deeper understanding of each component’s requirements and streamlined the development process by minimizing potential confusion and analysis errors.

The finished application meets its intended goals effectively, offering a simple yet functional user experience. Key features include user location detection, which prompts for location permissions and uses the device’s latitude and longitude to determine the user’s location and retrieve localized weather data. It also provides a 5-day forecast with corresponding weather animations that enhance the user experience. The forecast data is compiled into an engaging, user-friendly menu interface, demonstrating mindful design considerations. While the application is straightforward, its design allows for future enhancements such as location searches, making it an excellent foundation for more advanced functionality.

Throughout the project, I learned several valuable lessons. Understanding how to request and handle user permissions was crucial for accessing secured device data like location information. Adding animations demonstrated how even simple design elements can significantly enhance the appearance and interactivity of an application. Additionally, I learned the importance of filtering information from API responses within the data model to avoid overloading users with unnecessary data, ensuring that the display remains clean and user-friendly.

This project did a great job at demonstrating my ability to work with live APIs, design intuitive and somewhat visually appealing and navigable interfaces, and adhere to project requirements to deliver a polished application. It also reinforced the importance of thoughtful app design and its impact on user experience. Overall, the Slick Weather App project served as an excellent introduction to integrating real APIs into app development and showcased the value of clear and understandable design, paired with efficient data presentation in functional, user-centered applications.

                    Future getWeather(double lat, double lon) async {

                        }
                      } catch (e) {

                    }

Functionality to retrieve the API data using the latitude and longitude of the users location.                     /// Fetch the current location and detect the city name.
                    Future getCurrentCity() async {
                        try {
                        // Get the current position

                        // Fetch the city from the coordinates
                        return await getCityFromCoordinates(

                        print('Error in getCurrentCity: $e');
                        return "Error fetching city: $e";
                        }
                    }

Functionality to check for latitude and longitude positions, if they exist, the command line will display the location values and the will call a function to get the city using the position variables. If the position variables do not exist, the function will return an error. 


                
                    /// Detect city name from latitude and longitude from another API.
                    Future getCityFromCoordinates(double lat, double lon) async {
                      try {
                        final uri = Uri.parse(
                            'https://nominatim.openstreetmap.org/reverse?lat=$lat&lon=$lon&format=json');
                        final response = await http.get(uri);
                  
                        if (response.statusCode == 200) {
                          final data = jsonDecode(response.body);
                          return data['address']['city'] ?? "Unknown City";
                        } else {
                          return "Error fetching city";
                        }
                      } catch (e) {
                        print('Error in getCityFromCoordinates: $e');
                        return "Error fetching city";
                      }
                    }

Functionality that will try to call a city API that will try to return a city name when provided with valid latitude and longitude positions. After waiting for the response, it is checked for a status code of 200, meaning success, and if so, the city name is extracted from the response body. If the status code is not 200, it will return an error message. This is contained in a try -catch block to handle larger potential errors. 


                
                    /// Helper method to get the user's current position.
                    Future _getCurrentPosition() async {
                        try {
                        // Ensure location permissions are granted.
                        LocationPermission permission = await Geolocator.checkPermission();
                        if (permission == LocationPermission.denied) {
                            permission = await Geolocator.requestPermission();
                        }
                        if (permission == LocationPermission.deniedForever) {
                            throw Exception('Location permissions are permanently denied.');
                        }

                        // Get the user's current position.
                        return await Geolocator.getCurrentPosition();
                        } catch (e) {
                        print('Error in _getCurrentPosition: $e');
                        throw Exception('Error fetching current location: $e');
                        }
                    }

Functionality that will try to call the Geolocator package to get the user's current position, the function will first prompt the user for permission to access its location data. Upon approval, the function will use the Geolocator to pull the latitude and longitude of the users device from its location data and return the values. If the user denies the permission, it will throw a corresponding error message as well as fail the try - catch block.
